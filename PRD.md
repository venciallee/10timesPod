# 10timesPod — 双语字幕播客播放器 PRD

## 1. 产品概述

### 1.1 产品名称
**10timesPod** — 支持英文转录与中文翻译的播客 Web 播放器

### 1.2 产品定位
面向英文听力水平有限但希望通过播客学习英文内容的中文用户，提供一个带有**实时双语字幕（英文原文 + 中文翻译）**的播客收听体验。用户可以像看带字幕的视频一样听播客，大幅降低理解门槛。

### 1.3 目标用户
- 英文播客爱好者，但听力能力有限，需要文字辅助
- 想通过播客内容学习英文的用户
- 希望快速获取英文播客核心内容的中文读者

### 1.4 核心价值主张
> "听英文播客，看中文字幕" — 让每一个中文用户都能无障碍享受英文播客内容。

### 1.5 竞品参考
- **Snipd**: AI 生成转录、摘要、章节，支持 26 种语言 AI 但不提供中文逐句翻译字幕
- **Apple Podcasts**: 有转录功能但无翻译
- **小宇宙**: 中文播客为主，不支持英文转录翻译

**我们的差异化**: 专注"英文播客 + 中文字幕"这一垂直场景，提供逐句对照的沉浸式双语体验。


## 2. 功能需求

### 2.1 MVP 核心功能

#### F1: 播客订阅与管理
- **RSS 订阅**: 用户通过粘贴 RSS Feed URL 订阅播客
- **热门播客目录**: 内置热门英文播客推荐列表（如 Lex Fridman, Huberman Lab, Tim Ferriss 等），用户可一键订阅
- **订阅列表**: 展示所有已订阅播客，包含封面、名称、最新一期信息
- **单集列表**: 点击播客查看所有单集，按发布时间倒序排列，显示标题、时长、发布日期

#### F2: 播客播放器
- **音频播放**: 播放/暂停、进度条拖拽、快进/快退（15秒）
- **播放速度**: 支持 0.5x / 0.75x / 1x / 1.25x / 1.5x / 2x
- **底部常驻播放栏**: 切换页面时不中断播放
- **播放记忆**: 记录上次播放位置，下次打开自动续播

#### F3: 英文转录（Transcript）
- **AI 语音转文字**: 使用 Whisper 等模型将播客音频转为英文文本
- **逐句时间戳**: 每句话带有精确的开始/结束时间
- **字幕同步**: 播放时自动高亮当前正在播放的句子，自动滚动
- **点击跳转**: 点击任意一句文字，音频跳转到对应位置

#### F4: 中文翻译脚本
- **预生成翻译（优先）**: 订阅播客后，后台自动对英文转录进行中文翻译并缓存
- **实时翻译（兜底）**: 如果预生成翻译未就绪，播放时实时调用 LLM API 翻译当前句子
- **双语对照显示**: 英文原文和中文翻译逐句对照排列
- **显示模式切换**:
  - 仅英文
  - 仅中文
  - 英中对照（默认）

#### F5: 模型设置页（Settings）
- **多模型提供方支持**: 用户可自行配置 AI 服务的提供方和 API Key，不绑定单一厂商
- **支持的提供方（MVP）**:
  - **OpenRouter**: 聚合多模型网关，支持 OpenAI / Claude / Gemini / Llama 等，推荐默认选项
  - **OpenAI 直连**: 直接使用 OpenAI API（Whisper 转录 + GPT 翻译）
  - **自定义 OpenAI 兼容端点**: 用户填写 Base URL + API Key，适配各种兼容接口（如本地 Ollama、Azure OpenAI、国内代理等）
- **配置项**:
  - 提供方选择（下拉）
  - API Key 输入（密文显示，本地存储）
  - Base URL（仅自定义端点时填写）
  - 转录模型选择（如 whisper-large-v3）
  - 翻译模型选择（如 gpt-4o-mini, claude-sonnet, deepseek-chat 等）
  - 连接测试按钮：验证 API Key 是否有效
- **API Key 安全**: Key 仅存储在用户浏览器本地（localStorage 加密存储），不上传服务器；所有 AI 调用通过后端代理转发，前端不直接暴露 Key
- **默认配置**: 首次使用提供官方试用额度或引导用户配置自己的 Key

#### F6: 用户系统（轻量）
- **注册/登录**: 邮箱注册 或 第三方登录（Google）
- **个人订阅管理**: 绑定用户的订阅列表和播放进度
- **播放历史**: 记录最近收听的单集


### 2.2 P1 功能（第二阶段）

#### F7: 搜索与发现
- 按关键词搜索播客（集成 iTunes Search API / PodcastIndex API）
- 按分类浏览（科技、商业、教育、健康等）
- 热门排行榜

#### F8: AI 增强功能
- **单集摘要**: AI 生成每集的中文摘要（3-5 个要点）
- **章节导航**: AI 自动识别话题转换，生成章节标题
- **关键词高亮**: 自动识别并高亮专业术语，提供中英解释

#### F9: 学习功能
- **生词标记**: 用户可标记不认识的单词，生成个人词汇表
- **逐句跟读**: 点击句子可重复播放该句，练习听力

### 2.3 P2 功能（远期）
- 移动端 App（iOS/Android）
- 离线下载与离线字幕
- 社区功能（笔记分享、评论）
- 导出笔记到 Notion / Obsidian


## 3. 用户体验设计

### 3.1 核心页面

| 页面 | 说明 |
|------|------|
| 首页 / 发现页 | 展示热门推荐播客、已订阅播客的最新更新 |
| 播客详情页 | 展示某个播客的信息和单集列表 |
| 播放页 | 核心页面：音频播放器 + 双语字幕滚动区 |
| 订阅管理页 | 管理所有已订阅的播客 |
| 模型设置页 | 配置 AI 模型提供方、API Key、选择转录/翻译模型 |
| 个人中心 | 登录状态、播放历史、设置 |

### 3.3 设计原则
- **阅读优先**: 字幕区域占据主要空间，播放控件精简紧凑
- **无干扰**: 播放时字幕自动滚动跟随，用户无需手动操作
- **双语清晰**: 英文和中文用不同字体大小/颜色区分，一眼可辨
- **响应式**: 适配桌面和平板浏览器


## 4. 非功能需求

### 4.1 性能
- 音频加载首次播放延迟 < 3 秒
- 字幕同步延迟 < 500ms
- 实时翻译响应时间 < 2 秒/句

### 4.2 可用性
- 支持主流浏览器（Chrome, Safari, Firefox, Edge）
- 首屏加载 < 2 秒（桌面网络环境）

### 4.3 数据安全
- 用户数据加密存储
- API Key 不暴露于前端
- 支持 HTTPS

### 4.4 成本控制
- Whisper 转录: 约 $0.006/分钟（OpenAI API）
- LLM 翻译: 按 token 计费，预生成可批量处理降低成本
- 长期考虑引入本地模型（Whisper local + 开源翻译模型）降本


## 5. 业务指标

| 指标 | MVP 目标 |
|------|----------|
| 可订阅播客数 | 支持任意 RSS + 预置 20+ 热门播客 |
| 转录准确率 | > 95%（Whisper large-v3） |
| 翻译质量 | 通顺自然，专业术语准确 |
| 字幕同步精度 | 句级别同步，误差 < 1 秒 |
| 首屏加载 | < 2 秒 |


## 6. 里程碑计划

| 阶段 | 内容 | 周期 |
|------|------|------|
| **M1: 基础播放** | RSS 订阅 + 音频播放 + 英文转录展示 | 2 周 |
| **M2: 中文翻译** | 预生成翻译 + 实时翻译兜底 + 双语对照 | 2 周 |
| **M3: 体验完善** | 用户系统 + 播放记忆 + 热门推荐 + UI 打磨 | 2 周 |
| **M4: AI 增强** | 摘要 + 章节 + 搜索发现 | 3 周 |


## 7. 风险与应对

| 风险 | 影响 | 应对 |
|------|------|------|
| 转录成本随用户增长快速上升 | 高 | 引入本地 Whisper 模型 + 转录结果缓存共享 |
| 部分播客 RSS 不含完整音频 URL | 中 | 兼容多种 RSS 格式，做好异常处理 |
| LLM 翻译质量不稳定 | 中 | 选用高质量模型 + 后期引入人工校对机制 |
| 音频跨域访问限制 | 中 | 后端代理音频流 |
